Overview

What is CentraAssist?

CentraAssist is an internal AI-powered productivity platform built for CentraLogic employees. It gives teams a single workspace to ask AI questions, write and run code, preview web builds, and collaborate on shared projects — removing constant context-switching between separate tools.

Employees assigned to a project get shared access to its codebase and repository, enabling real-time collaboration without duplicating environments.

The problem

Fragmented tools, broken focus

CentraLogic teams relied on a patchwork of disconnected tools — separate apps for AI assistance, coding, file access, and planning. This created three core friction points:

01
Context switching overload

Developers moved between an AI chat, their IDE, and a file browser constantly, losing thread and momentum with every switch.

02
No shared project intelligence

AI conversations and code existed in personal silos. When multiple employees worked on the same project, there was no way to share context or work in tandem.

03
Generic tools, specific needs

Off-the-shelf AI tools had no understanding of internal workflows — no code assistant mode, no QA helper, no story planner tuned for how the team actually worked.

Key screens

The interface, screen by screen

Login & home

The login screen keeps authentication minimal — email-only entry reduces friction for internal users. The home screen greets the user by name with a natural-language prompt input. Quick-action chips (Check-In / Check-Out) surface the most common daily workflows with zero typing required.

Login — email-first, minimal friction
Home — personalised greeting + quick actions

Main chat workspace

The three-column layout is the core interaction model. Left sidebar holds conversation history. The central panel is the active AI conversation — it renders formatted code blocks with copy, edit, and share actions, and tables with an Export button. The right Tools panel offers Code Assistant, Story Planner, Friendly Assistant, and QA Helper as toggleable modes.

Three-column workspace: sidebar history · AI chat with code block · tools panel
AI-generated table with Export action — structured data stays usable, not just readable
Code editor — expanded mode

Code editor & preview

Expanding a code block opens a full split-panel editor. The left half shows editable code with line numbers; the right half keeps the conversation visible so context is never lost. A Run button executes inline, and a Preview tab renders web output — all without leaving the tool.

Code panel alongside active chat conversation
Preview tab — live web output rendered next to chat
File explorer panel — browse full project repository assigned to the user

Design decisions

Why it is designed the way it is

1

Three-column layout rather than full-screen chat

Most AI tools hide context the moment you act on a response. Keeping history, conversation, and tools all visible matches how developers actually work: referencing past output while building something new.

2

Code editor as an expansion of the chat, not a separate tab

Opening a code panel does not navigate away from the conversation. The split view preserves AI context on the right while the user edits on the left — closing the loop between suggestion and execution.

3

Quick actions as first-class UI elements on the home screen

Check-In and Check-Out are the highest-frequency daily actions. Surfacing them as chips above the prompt means zero typing for the most repeated task — saving real time across hundreds of daily uses.

4

Tools panel uses toggles, not separate pages

Tools are switched on/off inline rather than via navigation. This keeps the user in one workspace and signals that these are modifiers to AI behaviour, not separate products.

5

Login flow is email-only by design

Internal users authenticate via company email. Removing a password field reduces cognitive friction and sets the right expectation — this is a company tool, not a consumer product.

Reflection

What I learnt

Designing for power users requires density, not simplicity

The three-column layout might seem complex at first, but developers need information density. The lesson: simple for an expert often looks different from simple for a casual user.

Context preservation is an underrated UX principle

The biggest friction in AI tools is not the AI — it is losing your thread when you act on a response. The split-panel editor keeping conversation visible solved a real observed pain.

Internal tools deserve the same design rigour

With in-house products, the temptation is to ship good enough. CentraAssist reinforced that colleagues notice every friction point too — the quality bar should be equally high.

Mode-switching via toggles vs. navigation is a meaningful choice

Making tools toggleable rather than routable kept users in one mental model of the product — a small decision with a large effect on how unified the experience felt.

The best tools don't ask for attention — they just let you get the work done.
